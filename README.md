## 工作原则

1. 任何新增文档先放入 `docs/` 对应分区。
2. 原始素材只放 `assets/raw/`，不要直接改动。
3. 处理后的素材放 `assets/processed/` 或更具体的子目录。
4. 页面、数据、组件开发开始前，先补齐对应设计或数据说明。
5. 多模型协作时，每次交接都在 `docs/06-handoff/` 留一份任务说明或交接记录。
