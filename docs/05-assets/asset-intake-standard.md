# 素材处理规范

## 1. 素材分类

本项目可能涉及以下素材：

- 地图素材：哈萨克斯坦、曼吉斯套州、阿克套区域、油田作业区示意。
- 工业素材：油田、计量设备、SCADA、能源设施、管线、泵站。
- 图标素材：设备状态、异常等级、AI、人工复核、审计链、报告。
- 截图素材：参考案例、竞品、飞书文档、过程截图。
- 品牌素材：能源部、01.AI、项目临时标识。

## 2. 素材进入流程

所有素材按以下流程进入项目：

```text
收集 -> 放入 assets/raw -> 记录来源 -> 判断用途 -> 处理加工 -> 放入 assets/processed 或专项目录 -> 写入素材清单
```

## 3. 来源记录

每个重要素材需要记录：

- 文件名。
- 来源链接或提供人。
- 获取日期。
- 是否可公开使用。
- 当前用途。
- 是否需要替换。

素材清单后续放在：

```text
docs/05-assets/asset-inventory.md
```

## 4. 地图素材特别规则

地图类素材容易影响可信度，必须单独管理。

要求：

1. 原始地图放 `assets/raw/`。
2. 处理后地图放 `assets/maps/`。
3. 如果地图只用于示意，页面上不要暗示其具有精确 GIS 意义。
4. 区域名称、点位名称、油田名称必须和 mock 数据一致。
5. 不要直接使用带明显版权水印的地图。

## 5. 图片处理规则

处理后的图片建议：

- 页面背景图：优先 `.webp`。
- 图标和示意：优先 `.svg`。
- 截图留档：`.png`。
- 大图保留原始分辨率一份，再输出页面使用版本。

## 6. 禁止事项

- 不要把未经筛选的素材直接放进工程 `public/`。
- 不要把同一素材复制到多个目录。
- 不要覆盖原始素材。
- 不要在没有来源说明的情况下使用真实政府或企业标识。
- 不要使用会误导为真实监管数据的图片或地图。

## 7. 第一阶段建议

第一阶段先准备这些素材：

1. 阿克套/曼吉斯套州区域示意地图。
2. 油田作业区抽象底图或工业现场背景。
3. 设备状态图标。
4. 异常等级图标。
5. 审计链节点图标。

如果暂时没有合适真实素材，可以先用简化矢量图和抽象工业背景，不影响主流程开发。
