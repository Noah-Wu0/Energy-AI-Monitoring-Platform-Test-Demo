# 态势感知与能源运行看板参考

更新时间：2026 年 5 月 18 日

用途：为 1.1 全国能源设施实时态势图提供外部参考。这里记录的是信息结构和产品逻辑启发，不是视觉照搬。

## 1. EIA Real-time Operating Grid

链接：https://www.eia.gov/electricity/gridmonitor/dashboard

参考价值：

- 官方能源数据看板，适合参考“实时系统运行”而非营销式展示。
- 重点不是炫酷地图，而是把当前系统运行状态、区域和数据更新放在一起。

对 1.1 的启发：

- 顶部必须有“当前状态”和“最后刷新时间”。
- 地图和指标要服务一个问题：现在系统是否正常？
- 全国入口不宜塞满故事，故事应由事件流和下钻承接。

## 2. ERCOT Grid and Market Conditions

链接：https://www.ercot.com/gridmktinfo/dashboards

参考价值：

- ERCOT 将多个运行看板组织成一个“当前条件”入口，包括供需、系统状态、实时价格、停运、燃料结构、储能、风光、需求、天气等模块。
- 页面强调每个模块的更新时间，适合能源监管场景。

对 1.1 的启发：

- 我们的首页也应是“入口”，不是单一图表页。
- 右侧事件流和顶部 KPI 要有时间戳，体现实时监管。
- 油气、电力、供暖可以是同一入口的不同切换，不必做成三个孤立系统。

## 3. CAISO Today’s Outlook

链接：https://www.caiso.com/todays-outlook

参考价值：

- 电力系统运营面向公众公开当前供需、净负荷、排放和资源结构。
- 看板用多个稳定模块组织“今天系统如何运行”。

对 1.1 的启发：

- 第一屏应避免只讲空间关系，还要出现“当前运行态势”。
- 可用小型趋势条表达过去 24 小时变化，底部时间轴可承接这一点。

## 4. ENTSO-E Transparency Platform and Grid Map

链接：https://www.entsoe.eu/data/

参考价值：

- ENTSO-E 同时提供透明数据平台和电网地图。其电网地图并非真实 GIS 精度，而是展示输电网络和大型发电站的示意。
- 这对我们很重要：演示图可以是监管态势示意，不必承诺真实管线精度。

对 1.1 的启发：

- 地图上油气链路可以采用“监管示意网络”，但要标明是 mock / demo。
- 节点和链路关系比地理精度更重要。
- 全国地图可以保留真实边界，设施网络用示意化表达。

## 5. NESO real-time data dashboard

链接：https://www.neso.energy/news/national-grid-esos-real-time-data-dashboard-shows-britains-progress-decarbonising-electricity

参考价值：

- NESO 介绍其实时数据看板时，强调实时电力结构、历史视图和系统平衡动作透明度。
- 其叙事不是“我们有 dashboard”，而是“系统运营更透明、动作更可解释”。

对 1.1 的启发：

- 这一页讲述重点应是总部监管透明度，而不是地图视觉本身。
- 事件流应解释“为什么变黄/变红”，不能只给状态灯。

## 对 1.1 的综合设计结论

1. 首页默认展示全国地图，默认能源类型为油气，电力和供暖作为切换项。
2. 第一屏讲三句话：总部全局可见、异常可定位、处置可进入。
3. 地图不是装饰，它要表达设施、链路、区域和事件之间的关系。
4. 右侧事件流要承接后续场景，尤其是进入 1.2 设备档案和 2.1 异常检测中心。
5. 风格应是国家能源运行指挥入口：克制、清晰、可信、可审计，不做科幻大屏。
