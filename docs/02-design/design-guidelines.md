# 设计规范草案

## 1. 设计定位

本 demo 是能源监管 B 端界面，不是营销页，也不是科幻大屏。视觉风格应体现“监管可信、工业现场、数据可追溯、操作可控”。

关键词：

- 克制
- 清晰
- 可信
- 高信息密度
- 强状态表达
- 支持下钻和复核

## 2. 页面结构原则

页面按监管事件流程组织：

```text
监控态势 -> 异常详情 -> AI 事件详情 -> 人工复核 -> 审计链 -> 报告生成
```

每个页面必须回答三个问题：

1. 当前发生了什么？
2. 依据是什么？
3. 用户下一步能做什么？

## 3. 信息层级

建议每页采用三层信息：

1. 顶层：当前状态和关键结论。
2. 中层：图表、证据、解释、对象详情。
3. 底层：操作、审计、引用、历史记录。

不要把所有信息平铺在一个 dashboard 里。B 端界面的密度来自结构，不来自堆满屏幕。

## 4. 颜色原则

避免单一蓝黑科技风。建议采用：

- 主背景：低饱和浅灰或深灰，不使用大面积纯黑。
- 主色：能源监管蓝或青蓝，用于导航、焦点和主按钮。
- 风险色：红、橙、黄、绿固定对应严重程度。
- 审计色：AI 自动、人工确认、人工修改、系统生成分别使用稳定色。

状态色建议：

| 状态 | 颜色方向 | 用途 |
|---|---|---|
| 正常 | 绿 | 设备在线、数据正常 |
| 提示 | 黄 | 轻微偏离、待观察 |
| 重要 | 橙 | 需要复核 |
| 紧急 | 红 | 高风险异常 |
| 人工确认 | 蓝 | 已有人在回路确认 |
| 系统生成 | 灰 | 自动记录、报告草稿 |

## 5. 图表原则

图表必须服务判断：

- 能耗异常页以时序曲线和预测带为主。
- 不使用过多饼图。
- 不用装饰性雷达图作为核心判断依据。
- 异常区间要直接高亮。
- 图表旁边必须有自然语言解释和证据入口。

## 6. 地图原则

地图用于建立区域和对象关系，不是页面唯一主角。

第一阶段可以使用：

- 阿克套/曼吉斯套州简化区域示意图。
- 作业区点位和状态标识。
- 右侧事件流和下钻卡片。

暂不要求真实 GIS 精度。地图素材需要标明来源和处理状态。

## 7. 交互原则

- 关键操作必须明确反馈。
- 人工复核必须要求填写理由。
- 驳回或修改 AI 判断要记录审计日志。
- 模拟导出、模拟提交等 demo 行为要在状态上给出反馈。
- 不做无意义 hover 动效。

## 8. 文案原则

页面文案以中文为主，后续预留英文和哈萨克语。

文案风格：

- 短句。
- 监管语境。
- 不夸大 AI 能力。
- 不写“AI 已确认违规”。
- 使用“AI 初判”“建议核查”“需人工确认”等表达。

## 9. 组件优先级

第一阶段优先组件：

1. 页面框架和导航。
2. 状态 KPI 条。
3. 事件卡片。
4. 时序曲线图。
5. AI 解释面板。
6. 证据列表。
7. 人工复核表单。
8. 审计时间线。
9. 报告预览。

## 10. 验收要求

- 页面主线能连续点击。
- 不出现纯展示但无动作入口的页面。
- 图表、状态、证据和操作之间关系清楚。
- 领导层看得懂大意，技术部门能看到结构。
- 页面文字不遮挡、不溢出、不压缩到难读。
