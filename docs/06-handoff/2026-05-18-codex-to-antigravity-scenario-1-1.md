## 1. 当前结论

当前 React 页面已经具备基础工程结构、全国油气数据骨架和可运行页面，但**不应作为最终视觉方向继续小修小补**。

用户最新判断非常明确：

- 地图太小，视觉中心不够突出。
- 地图背景和呈现形式不对，看起来乱。
- 信息层级没有真正拉开，部分模块重复。
- 地图应该位于中间偏上。
- 地图下方可以放 2-3 组卡片，用来讲清楚内容。
- 左右两侧面板底部应与下方区域对齐。
- 卡片内容要重新梳理，不能混乱。
- “下钻处置/报警”类卡片应该是告警色，不应是绿色。

因此，接手时建议按“重做布局和地图表达”的方式处理，而不是继续沿用当前视觉细节。

## 2. 用户已认可的方向

用户认可过的总体方向是：

**方案 A + C：2.5D 全国沙盘 + 产业链走廊**

含义：

- 中间地图是主舞台，不是普通行政区底图。
- 地图可以适当倾斜，有沙盘感，但不能乱。
- 主产业链不应是细线乱连，应做成清晰的“走廊带”或主通道。
- 全国层级看州级风险和产业链走向，不直接铺满油田细节。
- 下一级再进入州、企业、作业区、计量点和事件证据包。

## 3. 这页真正要回答的问题

这页不是为了展示“很多节点”，也不是全国能源百科。

它应该回答能源部总部进入系统后的第一问题：

> 全国能源系统现在是否正常？如果不正常，问题在哪个州、哪条产业链、影响什么节点、下一步谁去复核？

推荐页面主叙事：

```text
全国态势 -> 风险区域 -> 产业链影响 -> 当前事件 -> 下钻处置
```

## 4. 建议的信息层级

### 第一层：顶部 KPI

保留顶部 KPI，但建议进一步收敛为“强判断”：

- 在线/接入规模
- 当前异常
- 待人工复核
- 数据延迟/最近刷新

顶部不要承担复杂解释。

### 第二层：中间地图

中间地图必须是第一视觉，建议：

- 放在中间偏上。
- 地图面积更大。
- 背景更干净，不要出现大面积不必要渐变或留白。
- 行政区只做基础分区和风险底色。
- 主通道用 3-4 条粗走廊表达：
  - CPC / 西部主出口
  - UAS / 北向交接
  - 中哈管道 / 东向出口
  - 阿克套-BTC / 里海出口
- 只保留 2 个风险 callout：
  - 曼吉斯套州：港储运链路偏离
  - 西哈州：气凝析数据延迟

### 第三层：地图下方卡片

地图下方放 2-3 组卡片，不要过高。

建议三张：

1. **当前判断**
   - 说清“全国总体可见，风险集中在哪里”。

2. **主通道状态**
   - 用小型指标/条形图表达 4 条通道状态。

3. **下钻处置 / 待办**
   - 这是告警/待处置模块，颜色应为橙色或红橙色。
   - 不要用绿色图标或绿色标题。

### 第四层：左侧

左侧只做控制和索引：

- 能源类型切换：油气 / 天然气 / 电力 / 供暖
- 筛选器：地区 / 企业 / 等级
- 州级风险排行

不要在左侧放大段解释、产业链分层、节点详情，否则会重复。

### 第五层：右侧

右侧应是处置队列：

- 通道健康度
- 实时事件流

不要再放底部露半截的大 AI 初判卡。
